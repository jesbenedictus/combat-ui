Content grids

Use grid classes for feature lists, case cards, general card groups, logo strips, and stats. They are CSS-only, configurable with `data-*` modifiers and `--cui-*` variables, and work from CMS or static HTML.

API reference
Class Purpose
.cui-feature-grid / .cui-feature-card Feature lists with consistent icon, heading, and copy spacing.
.cui-case-grid / .cui-case-card / .cui-case-card-media / .cui-case-card-body / .cui-case-card-meta Case-study cards with optional media, body, and meta line.
.cui-card-grid / .cui-content-card General-purpose card grid for resource lists and article tiles.
.cui-logo-strip / .cui-logo-item Logo / partner strip with consistent block-size cells.
.cui-stats-grid / .cui-stat / .cui-stat-value / .cui-stat-label Number-focused grids for metrics, proof points, and KPIs.
Modifier Purpose
data-columns="2 | 3 | 4" Sets the column count for any of the grid wrappers.
data-gap="compact | spacious" Tightens or relaxes the grid gap.
data-align="start | center" On a feature card or stat block — aligns its inner content.
data-variant="flat" On a case card — drops the elevation shadow.
data-variant="borderless" On a content card — drops the surrounding border for editorial layouts.
Variable Default
--cui-feature-icon-size --cui-size-control
--cui-case-card-media-ratio Per-instance aspect ratio override (e.g. 4 / 3).
--cui-content-grid-gap --cui-space-4
--cui-logo-item-min-block-size 4rem
--cui-stat-value-size --cui-font-size-5
