Changelog

Every notable change to the Combat UI public API — custom elements and their attributes, slots, events, and methods; light-DOM CSS classes and data-* hooks; exported helpers; and package subpath exports. Internal refactors, build tooling, and dev-dependency bumps are intentionally left out. Releases follow semantic versioning; while the project is pre-1.0, minor versions may include breaking changes, which are always flagged below with an upgrade guide.

v0.5.0

Breaking

CSS footprint reduction. The block stylesheets stop re-implementing structural chrome and instead compose the foundational primitives (.cui-surface, .cui-stack, .cui-cluster, .cui-grid) in markup. Dozens of bespoke *-body, *-actions, *-grid, and *-meta subclasses are removed in favour of a small canonical vocabulary. Removals and the migration guide below are filled in as each block lands.

Removed

  • .cui-cta-actions - the CTA actions row is now a plain .cui-cluster. Its default gap (--cui-space-3) is identical, so the swap is purely a class rename in markup.
  • The five typed content grids - .cui-card-grid, .cui-feature-grid, .cui-case-grid, .cui-stats-grid, .cui-logo-strip (and their --cui-*-grid-min / --cui-content-grid-* vars). Use .cui-grid with --cui-grid-min per content type (18rem features, 20rem cases, 10rem stats, 9rem logos); the card classes (.cui-feature-card, .cui-case-card, .cui-content-card, .cui-stat, .cui-logo-item) are unchanged.
  • .cui-testimonial-grid (and its --cui-testimonial-grid-min var) - lay testimonial cards out with .cui-grid and --cui-grid-min: 18rem.
  • .cui-testimonial-author-text - the name + role pair is now a plain .cui-stack with --cui-stack-gap: var(--cui-space-1).
  • .cui-person-body - the person body column is now a .cui-stack; the card sets its gap from --cui-person-body-gap (preserved) and pins min-inline-size: 0 for row orient.
  • .cui-person-links - the links row is now a .cui-cluster. The card flattens the <ul> and adds the gap above it, and .cui-person a styles the links (compact, bold, accent on hover) without the dedicated class.
  • .cui-team-grid (and its --cui-team-grid-min var) - lay person cards out with .cui-grid (default --cui-grid-min: 16rem). For the leadership layout set --cui-grid-min: 20rem + data-gap="spacious" and give each card data-variant="leadership" (the grid no longer scales its children).
  • .cui-vacancy-card-body - the card body is now a .cui-stack; a scoped rule feeds its gap from --cui-vacancy-card-gap and carries the card padding (the card zeroes its own so media keeps bleeding edge-to-edge).
  • .cui-vacancy-card-actions - the actions row is now a .cui-cluster (identical --cui-space-3 gap); a scoped rule keeps it bottom-aligned with the small top padding.
  • .cui-vacancy-meta / .cui-vacancy-meta-item (and the plain variant) - the fact row is now the shared .cui-meta; add data-variant="chips" for the rounded-pill look (plain .cui-meta is the old plain variant).
  • .cui-vacancy-grid (and its --cui-vacancy-grid-min var) - lay cards out with .cui-grid and --cui-grid-min: 20rem; the data-columns / data-gap axes carry over.
  • .cui-vacancy-list (and its --cui-vacancy-list-gap var) - the single-column overview is a .cui-stack (set --cui-stack-gap for the compact / spacious rhythm). The dense data-variant="stacked" index moves to the stack and still draws hairline separators between vacancy cards.
  • .cui-vacancy-eyebrow (the detail-header eyebrow) - it was identical to .cui-eyebrow; use that canonical class directly in markup.
  • .cui-vacancy-card-media, .cui-vacancy-card-eyebrow, .cui-vacancy-card-title, .cui-vacancy-card-tagline, .cui-vacancy-card-excerpt, .cui-vacancy-card-status - the vacancy card now styles its own descendants (contextual markup), so the slot classes are gone. The team label is a bare .cui-eyebrow, the title a .cui-display (link wrapped in a child <a>), the tagline a .cui-lead, the excerpt a bare <p>, the media the first-child <img> / <picture> / <a>, and the status a direct-child <span>. The card retunes the type primitives to card scale in place; the --cui-vacancy-card-* knobs are preserved.
  • .cui-vacancy-header, .cui-vacancy-title, .cui-vacancy-tagline, .cui-vacancy-main, .cui-vacancy-section, .cui-vacancy-aside-title, .cui-vacancy-summary, .cui-vacancy-apply, .cui-vacancy-apply-note, .cui-vacancy-contact, .cui-vacancy-contact-person - the detail page now styles its own descendants (contextual markup). The title band is the detail's direct-child <header> (compose .cui-stack) carrying .cui-eyebrow / .cui-display / .cui-lead, retuned to page scale in place (the .cui-display 13ch hero measure cap dropped). The body column is a .cui-flow / .cui-prose child with plain <section>s. Inside .cui-vacancy-aside (kept): a .cui-eyebrow section label, a <dl> quick-facts list, a .cui-stack with the apply .cui-button, and a buttonless .cui-stack contact block (a <strong> name). The --cui-vacancy-title-size / --cui-vacancy-main-measure / --cui-vacancy-aside-top knobs are preserved.
  • .cui-page-intro-content, .cui-page-intro-title, .cui-page-intro-meta, .cui-page-intro-actions, .cui-page-intro-aside - the page-intro band now styles its own descendants (contextual markup). The content column is a .cui-stack, the title a .cui-display (retuned to page-intro scale in place, 16ch measure), the meta a .cui-meta (a <ul> of labels), the actions a .cui-cluster, and the summary rail a .cui-surface.cui-aside. The .cui-page-intro-layout grid wrapper and the --cui-page-intro-content-size / --cui-page-intro-title-size knobs are preserved.

Changed

  • .cui-cta no longer carries its own chrome - it composes .cui-surface (border, radius, shadow, background, padding) and .cui-stack (the content↔actions gap). Update markup from class="cui-cta" to class="cui-cta cui-surface cui-stack". The --cui-cta-padding and --cui-cta-gap knobs are preserved - they now feed --cui-surface-padding and --cui-stack-gap.
  • .cui-cta-content now composes .cui-stack; update it to class="cui-cta-content cui-stack". It keeps only the tightened gap and the content measure cap.
  • .cui-stack now zeroes margin-block on its direct children, so the gap is the sole source of vertical rhythm. This makes it a drop-in for the per-block *-body margin resets being removed across this release.
  • .cui-grid now supports the [data-columns="2|3|4"] and [data-gap="compact|spacious"] axes (ported from the removed typed grids), and aligns .cui-feature-card children across a row via subgrid.
  • The content cards - .cui-feature-card, .cui-content-card, .cui-stat, .cui-logo-item, .cui-case-card - no longer carry their own chrome; they compose .cui-surface. Add it in markup, e.g. class="cui-surface cui-feature-card". The --cui-content-card-* knobs are preserved - they now feed the matching --cui-surface-* override vars.
  • .cui-case-card-body now composes .cui-stack; update it to class="cui-case-card-body cui-stack". It keeps only the body padding and the tightened gap delta.
  • .cui-quote and .cui-testimonial no longer carry their own chrome; they compose .cui-surface. Add it in markup, e.g. class="cui-surface cui-quote". The --cui-quote-* knobs are preserved - they now feed the matching --cui-surface-* override vars.
  • .cui-person no longer carries its own chrome; it composes .cui-surface. Add it in markup, e.g. class="cui-surface cui-person" (including org-chart cards). The --cui-content-card-* / --cui-person-* knobs are preserved - they now feed the matching --cui-surface-* override vars.
  • .cui-vacancy-card no longer carries its own chrome; it composes .cui-surface and styles its own body descendants contextually. Add it in markup, e.g. class="cui-surface cui-vacancy-card" with a .cui-stack body. The card feeds the --cui-surface-* vars from its --cui-content-card-* / --cui-vacancy-card-* knobs and sets --cui-surface-padding: 0 so media still bleeds; the body carries the padding. The featured / compact / flat / borderless / data-status variants are unchanged. (The body slot classes are dropped - see Removed.)
  • .cui-vacancy-aside no longer carries its own chrome; it composes .cui-surface (muted background, no shadow) + .cui-stack and styles its own descendants. Add the primitives in markup, e.g. class="cui-vacancy-aside cui-surface cui-stack"; the summary / apply / contact internals no longer need slot classes (see Removed), and it keeps only its delta (the sticky rail, the full-width apply button) feeding its gap through --cui-stack-gap.

Deprecated

  • .cui-card - use the .cui-surface.cui-stack recipe instead. The class keeps working as a back-compat alias for one minor and will be removed in the next. Migrate class="cui-card" to class="cui-surface cui-stack" (add --cui-stack-gap: var(--cui-space-3) for the original 0.75rem gap).

Added

  • Override custom properties on .cui-surface so any block can re-skin the chrome without redeclaring it: --cui-surface-bg, --cui-surface-border, --cui-surface-radius, --cui-surface-shadow, and --cui-surface-padding (each defaults to the previous hardcoded value).
  • New .cui-meta class — one canonical metadata row (wrapping list with · separators and list reset) that replaces the per-block *-meta rows. Tunable via --cui-meta-gap and --cui-meta-separator.
  • .cui-meta gains a data-variant="chips" modifier — renders each item as a muted rounded pill (no separators), the shared replacement for the vacancy fact-pill row.
  • Override custom properties on the editorial type primitives so blocks can compose and retune them without redeclaring: --cui-display-size / -weight / -leading / -tracking / -measure on .cui-display, and --cui-lead-size / -leading / -measure on .cui-lead (each defaults to the previous hardcoded value).
  • New .cui-aside class — one canonical summary-rail layout (a start-aligned stack that collapses out of flow when empty). Compose .cui-surface for the card chrome and tune the internal rhythm via --cui-aside-gap; used by .cui-page-intro and available for any sidebar / fact-panel rail.

2026-06-24

v0.4.0

Breaking

Section primitives move out of the shadow DOM. cui-hero, cui-cta, and cui-page-intro are no longer custom elements — they are now light-DOM CSS blocks you compose from plain markup. cui-navbar and cui-tabs become progressive enhancements over light-DOM structure rather than slotted shadow roots. The payoff is server-renderable, CMS-friendly, fully styleable markup with no flash-of-undefined-element.

Removed

  • The cui-hero, cui-cta, and cui-page-intro custom elements, along with their exported classes (CuiHero, CuiCta, CuiPageIntro). They are replaced by the .cui-hero, .cui-cta, and .cui-page-intro light-DOM class blocks.
  • The @combat-ui/core/hero, @combat-ui/core/cta, and @combat-ui/core/page-intro subpath exports. Remove these imports — the styles ship with @combat-ui/core/styles.

Changed

  • cui-navbar now enhances light-DOM .cui-navbar-* markup instead of slotting brand/nav/actions into a shadow root. The expanded, sticky, sticky-offset, and sticky-z-index attributes are unchanged; the burger toggle is injected for you.
  • cui-tabs now reads its tabs and panels from light DOM: tabs live in a .cui-tablist as .cui-tab buttons, and panels are the remaining element children in order. The initial-selection signal changed from aria-selected="true" to data-selected="true". Keyboard support (Arrow / Home / End) and ARIA wiring are preserved.
  • The inverse boolean attribute on the former CTA / page-intro elements is now the data-tone="inverse" attribute on the corresponding block.
  • Variant and alignment are expressed with data-variant and data-align attributes instead of variant / align element attributes.

Added

  • Light-DOM block classes: .cui-hero (with .cui-hero-inner, .cui-hero-content, .cui-hero-media, .cui-hero-bg, .cui-hero-actions, .cui-hero-title), .cui-cta (with .cui-cta-content, .cui-cta-title, .cui-cta-actions), and .cui-page-intro (with .cui-page-intro-layout, -content, -title, -meta, -actions, -aside).
  • New public CSS custom properties on the hero block: --cui-hero-background-image and --cui-hero-background-position (set them on .cui-hero-bg for the background variant). The remaining --cui-hero-*, --cui-cta-*, and --cui-page-intro-* customization points carry over.

Upgrade guide

cui-hero → .cui-hero block. Replace the element with a <section class="cui-hero">, move slots to classed children, and wrap content in .cui-hero-inner / .cui-hero-content.

Before — v0.3.x

After — v0.4.0

For the background variant, the background-src / background-position attributes are gone. Add a .cui-hero-bg element and set --cui-hero-background-image / --cui-hero-background-position inline: <div class="cui-hero-bg" style="--cui-hero-background-image: url('hero.jpg'); --cui-hero-background-position: center;"></div>.

cui-cta → .cui-cta block. The variant attribute becomes data-variant (values unchanged: simple, split, full-bleed, sticky) and the inverse attribute becomes data-tone="inverse".

Before — v0.3.x

After — v0.4.0

cui-page-intro → .cui-page-intro block. Wrap the content (and optional aside) in a .cui-page-intro-layout grid; map slots to classes (slot="meta" → .cui-page-intro-meta, slot="aside" → .cui-page-intro-aside).

Before — v0.3.x

After — v0.4.0

cui-tabs markup. Group tabs in a .cui-tablist and mark the initial tab with data-selected="true"; panels are the remaining children, matched to tabs in document order.

Before — v0.3.x

After — v0.4.0

cui-navbar moves the same way — slots become a light-DOM .cui-navbar-inner / .cui-navbar-bar / .cui-navbar-collapse structure with .cui-nav, .cui-nav-link, and .cui-dropdown children. See the Navigation page for the full markup.

Breaking

Token rename and a lighter dependency footprint for the map.

Changed

  • Public custom property renamed: --cui-radius-control → --cui-control-radius. Update any override of the old name.
  • leaflet and leaflet.markercluster moved from dependencies to peerDependencies. If you use cui-map, install Leaflet alongside Combat UI.

Upgrade guide

  • Find-and-replace --cui-radius-control with --cui-control-radius in your stylesheets and inline overrides.
  • If your project renders cui-map, add Leaflet to your own dependencies: npm install leaflet leaflet.markercluster.
